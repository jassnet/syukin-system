# デプロイメント

## 概要

出退勤システムのデプロイ手順と環境設定について説明します。

## 前提条件

- Python 3.11以上
- PostgreSQL（本番環境推奨）またはSQLite（開発環境）
- Gunicorn（本番環境）

## ローカル開発環境

### セットアップ手順

```bash
# 1. 仮想環境作成
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 2. 依存パッケージインストール
pip install -r requirements.txt

# 3. 環境変数設定（オプション）
# .envファイルを作成して設定をカスタマイズ

# 4. DB初期化（初回のみ）
flask --app app init-db

# 5. アプリ起動
python -m flask --app app run -p 8000
# または
gunicorn -w 3 -b 0.0.0.0:8000 app:app
```

### 初期管理者ユーザー作成

```python
from app import app, db, User
with app.app_context():
    admin = User(username='admin', role='admin')
    admin.set_password('初期パスワード')
    admin.name = '管理者'
    db.session.add(admin)
    db.session.commit()
```

## 本番環境（PaaS）

### 対応PaaS

以下のPaaSプラットフォームでデプロイ可能です：

- **Heroku** - 最も一般的なPaaS（有料プラン必要）
- **Render** - 無料プランあり、PostgreSQL対応
- **Railway** - シンプルなデプロイ、PostgreSQL対応
- **Fly.io** - グローバル分散、PostgreSQL対応
- **その他** - Procfile対応のPaaSであれば動作可能

### デプロイ前の準備

#### 1. GitHubリポジトリへのプッシュ

```bash
# コードをGitHubにプッシュ（まだの場合）
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/syukin-system.git
git push -u origin main
```

#### 2. SECRET_KEYの生成

```bash
# Pythonで生成
python -c "import secrets; print(secrets.token_urlsafe(32))"

# またはPowerShellで生成（Windows）
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

生成された値をコピーして保存してください（後で環境変数として設定します）。

### デプロイ手順（共通）

#### ステップ1: PaaSアカウント作成とプロジェクト作成

選択したPaaSのアカウントを作成し、新しいプロジェクト/サービスを作成します。

#### ステップ2: GitHubリポジトリの連携

PaaSのダッシュボードから、GitHubリポジトリを連携します。

#### ステップ3: データベース（PostgreSQL）の作成

**重要**: 本番環境では必ずPostgreSQLを使用してください。

各PaaSでPostgreSQLデータベースを作成します：

- **Heroku**: Add-ons → Heroku Postgres
- **Render**: New → PostgreSQL
- **Railway**: New → Database → PostgreSQL
- **Fly.io**: `fly postgres create`

#### ステップ4: DATABASE_URLの取得

PostgreSQLデータベースを作成すると、**DATABASE_URL**が自動的に生成されます。

**DATABASE_URLの形式**:
```
postgresql://ユーザー名:パスワード@ホスト名:ポート/データベース名
```

例:
```
postgresql://user:password123@dpg-xxxxx-a.singapore-postgres.render.com:5432/syukin_db
```

**取得方法**:
- **Heroku**: `heroku config:get DATABASE_URL` またはダッシュボードのConfig Vars
- **Render**: ダッシュボードの「Internal Database URL」または「External Database URL」
- **Railway**: ダッシュボードの「Variables」タブ
- **Fly.io**: `fly postgres connect` またはダッシュボード

#### ステップ5: 環境変数の設定

PaaSのダッシュボードで以下の環境変数を設定します：

**必須環境変数**:

| 変数名 | 説明 | 例 |
|--------|------|-----|
| `SECRET_KEY` | セッション暗号化用の秘密鍵（ステップ2で生成した値） | `abc123...xyz` |
| `DATABASE_URL` | PostgreSQL接続URL（ステップ4で取得した値） | `postgresql://...` |
| `TIMEZONE` | 表示用タイムゾーン | `Asia/Tokyo` |

**推奨環境変数**:

| 変数名 | 説明 | 値 |
|--------|------|-----|
| `SESSION_COOKIE_SECURE` | HTTPS使用時は必須 | `true` |

**オプション環境変数**:

| 変数名 | 説明 | 既定値 |
|--------|------|--------|
| `CSV_EXPORT_MAX_DAYS` | CSVエクスポートの最大日数 | `365` |

#### ステップ6: ビルド・起動コマンドの設定

多くのPaaSでは`Procfile`を自動認識しますが、手動設定が必要な場合：

- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `gunicorn app:app`

#### ステップ7: デプロイ実行

GitHubにプッシュすると自動デプロイが開始されます。または、PaaSのダッシュボードから「Deploy」ボタンをクリックします。

#### ステップ8: データベースの初期化

デプロイが完了したら、データベーステーブルを作成します。

**方法1: PaaSのコンソール/シェルから実行**

```bash
# PaaSのコンソール/シェルに接続
flask --app app init-db
```

**方法2: ローカルから実行（DATABASE_URLを設定）**

```bash
# 環境変数を設定して実行
export DATABASE_URL="postgresql://..."
flask --app app init-db
```

**方法3: Pythonスクリプトで実行**

```python
from app import app, db
with app.app_context():
    db.create_all()
    print("データベースを初期化しました")
```

#### ステップ9: 初期管理者ユーザーの作成

データベース初期化後、初期管理者ユーザーを作成します。

**方法1: PaaSのコンソール/シェルから実行**

```bash
# PaaSのコンソール/シェルに接続してPythonを起動
python
```

```python
from app import app, db, User
with app.app_context():
    admin = User(username='admin', role='admin')
    admin.set_password('強力な初期パスワードを設定')  # 必ず変更してください
    admin.name = '管理者'
    admin.email = 'admin@example.com'  # オプション
    db.session.add(admin)
    db.session.commit()
    print("管理者ユーザーを作成しました")
```

**方法2: ローカルから実行（DATABASE_URLを設定）**

```bash
export DATABASE_URL="postgresql://..."
python
```

その後、上記のPythonコードを実行します。

#### ステップ10: 動作確認

1. アプリケーションのURLにアクセス（例: `https://your-app.onrender.com`）
2. ログインページが表示されることを確認
3. ステップ9で作成した管理者ユーザーでログイン
4. ダッシュボードが表示されることを確認

### 各PaaS別の詳細手順

#### Heroku

1. **Heroku CLIのインストール**: https://devcenter.heroku.com/articles/heroku-cli
2. **ログイン**: `heroku login`
3. **アプリ作成**: `heroku create your-app-name`
4. **PostgreSQL追加**: `heroku addons:create heroku-postgresql:mini`
5. **環境変数設定**:
   ```bash
   heroku config:set SECRET_KEY="your-secret-key"
   heroku config:set TIMEZONE="Asia/Tokyo"
   heroku config:set SESSION_COOKIE_SECURE="true"
   ```
6. **デプロイ**: `git push heroku main`
7. **DB初期化**: `heroku run flask --app app init-db`
8. **管理者作成**: `heroku run python` でPythonコンソールを起動し、上記のコードを実行

#### Render

1. **ダッシュボード**: https://dashboard.render.com
2. **New Web Service** をクリック
3. **GitHubリポジトリを選択**
4. **設定**:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app`
5. **New PostgreSQL** でデータベースを作成
6. **環境変数**を設定（DATABASE_URLは自動設定されます）
7. **Create Web Service** をクリック
8. デプロイ完了後、**Shell**からDB初期化と管理者作成を実行

#### Railway

1. **ダッシュボード**: https://railway.app
2. **New Project** → **Deploy from GitHub repo**
3. **リポジトリを選択**
4. **New** → **Database** → **Add PostgreSQL**
5. **Variables**タブで環境変数を設定
6. デプロイが自動開始されます
7. デプロイ完了後、**View Logs** → **Open Shell**からDB初期化と管理者作成を実行

#### Fly.io

1. **Fly CLIのインストール**: https://fly.io/docs/getting-started/installing-flyctl/
2. **ログイン**: `fly auth login`
3. **アプリ作成**: `fly launch`
4. **PostgreSQL作成**: `fly postgres create`
5. **PostgreSQL接続**: `fly postgres attach -a your-app-name`
6. **環境変数設定**: `fly secrets set SECRET_KEY="your-secret-key" TIMEZONE="Asia/Tokyo" SESSION_COOKIE_SECURE="true"`
7. **デプロイ**: `fly deploy`
8. **DB初期化**: `fly ssh console` → `flask --app app init-db`
9. **管理者作成**: `fly ssh console` → `python` で上記のコードを実行

### 環境変数の詳細

上記のステップ5で設定する環境変数の詳細です。

#### 必須環境変数

**SECRET_KEY**
- **説明**: セッション暗号化、CSRFトークン生成、HMAC署名に使用
- **生成方法**: `python -c "import secrets; print(secrets.token_urlsafe(32))"`
- **要件**: 32文字以上のランダムな文字列
- **注意**: 本番環境では必ず設定してください。設定しないと起動のたびに新しい値が生成され、セッションが無効になります

**DATABASE_URL**
- **説明**: PostgreSQLデータベースへの接続URL
- **形式**: `postgresql://ユーザー名:パスワード@ホスト名:ポート/データベース名`
- **取得方法**: 各PaaSでPostgreSQLを作成すると自動生成されます
- **注意**: 本番環境では必ずPostgreSQLを使用してください。SQLiteは開発環境専用です

**TIMEZONE**
- **説明**: 表示用のタイムゾーン
- **値**: `Asia/Tokyo`（日本時間の場合）
- **注意**: データベース内の時刻はUTCで保存されますが、表示時にこのタイムゾーンで変換されます

#### 推奨環境変数

**SESSION_COOKIE_SECURE**
- **説明**: HTTPS使用時にセッションCookieを保護
- **値**: `true`（HTTPS使用時）、`false`または未設定（HTTP使用時）
- **注意**: 本番環境でHTTPSを使用する場合は必ず`true`に設定してください

#### オプション環境変数

**CSV_EXPORT_MAX_DAYS**
- **説明**: 管理画面から手動でCSVエクスポートできる最大日数
- **既定値**: `365`（1年）
- **例**: `CSV_EXPORT_MAX_DAYS=730`（2年間）

### Procfile

**ファイル位置**: `Procfile`

```
web: gunicorn app:app
```

### Gunicorn設定

**推奨設定**:
```bash
gunicorn -w 4 -b 0.0.0.0:$PORT --timeout 120 app:app
```

**パラメータ説明**:
- `-w 4`: ワーカー数（CPUコア数×2+1を推奨）
- `-b 0.0.0.0:$PORT`: バインドアドレスとポート
- `--timeout 120`: タイムアウト（秒）

## データベース移行

### SQLite → PostgreSQL

1. **データエクスポート**:
   ```bash
   sqlite3 attendance.db .dump > dump.sql
   ```

2. **PostgreSQLインポート**:
   - テーブル構造を確認
   - データをインポート（必要に応じて変換）

3. **接続URL設定**:
   ```
   DATABASE_URL=postgresql://user:password@host:port/dbname
   ```

### スキーマ変更

現バージョンではAlembicなどのマイグレーションツールは使用していません。

**手動マイグレーション手順**:
1. バックアップ取得
2. 新しいカラムを追加（`ALTER TABLE`）
3. データ移行（必要に応じて）
4. アプリ再起動

- [ ] 不要なポートが閉じられている

## パフォーマンス最適化

### データベース

- **インデックス**: 必要に応じて追加
- **接続プール**: SQLAlchemyの設定で調整
- **クエリ最適化**: N+1問題の回避

### アプリケーション

- **Gunicornワーカー数**: CPUコア数に応じて調整
- **キャッシュ**: 必要に応じてRedis等を導入
- **静的ファイル**: CDNやNginxで配信

- **アプリケーションログ**: Flask標準ロガー

- **監査ログ**: `audit_logs`テーブル

## トラブルシューティング

### よくある問題

1. **データベース接続エラー**
   - `DATABASE_URL`が正しいか確認
   - PostgreSQLが起動しているか確認
   - 接続権限を確認

2. **ログインできない**
   - 初期管理者ユーザーが作成されているか確認
   - パスワードが正しいか確認

3. **CSVエクスポートが失敗する**
   - 指定期間が `CSV_EXPORT_MAX_DAYS` を超えていないか確認
   - DB接続が安定しているか確認

4. **パフォーマンスが悪い**
   - データベースインデックスを確認
   - Gunicornワーカー数を調整
   - クエリを最適化
